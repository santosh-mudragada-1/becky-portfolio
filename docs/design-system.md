
# Design System

## Design Philosophy

The website should feel like a product built by a creative Product Manager.

Not a corporate portfolio.

Not a designer portfolio.

It should be playful, expressive, interactive, and memorable while remaining polished and professional.

Every visual element should encourage exploration.

---

# Art Direction

The visual language should combine:

* Product management
* Marketing
* Creative workspace
* Hand-drawn doodles
* Modern web interactions

Imagine a workspace filled with:

* Sticky notes
* Whiteboards
* Coffee cups
* Charts
* Kanban boards
* User journey maps
* Product roadmaps
* Analytics dashboards
* Sketches

The website should feel handcrafted.

---

# Visual Personality

Keywords:

* Bright
* Playful
* Creative
* Quirky
* Friendly
* Energetic
* Expressive
* Slightly chaotic
* Smart
* Modern

Avoid:

* Minimal
* Corporate
* Luxury
* Dark enterprise dashboards
* Generic portfolio layouts

---

# Color System

The primary color should be configurable.

Supported themes:

🟡 Mango (Default)

🟠 Coffee

🟣 Grape

🔵 Ocean

🟢 Matcha

The rest of the UI should automatically adapt to the selected theme.

---

# Typography

Style:

* Friendly
* Rounded
* Highly readable
* Modern

Headings

* Large
* Bold
* Playful

Body

* Clean
* Comfortable spacing
* Easy to scan

Avoid long paragraphs.

---

# Border Radius

Rounded corners throughout the interface.

Cards should feel approachable.

Avoid sharp edges.

---

# Shadows

Use soft layered shadows.

No harsh drop shadows.

Floating cards should feel lightweight.

---

# Icons

Style:

* Rounded
* Friendly
* Consistent

Use Lucide icons where possible.

Illustrations should complement icons.

---

# Illustrations

Illustrations should appear throughout the website.

Examples:

* Coffee mugs
* Sticky notes
* Product roadmaps
* Whiteboards
* Charts
* Doodles
* User personas
* Tiny characters
* Hand-drawn arrows

Illustrations should feel fun rather than decorative.

---

# Buttons

Buttons should feel interactive.

Examples:

Primary

Explore Projects

Secondary

Read Case Study

Ghost

Maybe Later

Hover states should always provide feedback.

---

# Cards

Reusable cards should include:

* Project Card
* Skill Card
* Tool Card
* Dashboard Card
* Achievement Card
* Sticky Note
* Timeline Card

Every card should animate on hover.

---

# Animations

Animations should feel:

* Smooth
* Organic
* Playful
* Purposeful

Never distracting.

Recommended durations:

Fast: 150–200ms

Medium: 300–500ms

Large transitions: 600–900ms

---

# Microinteractions

Every interaction should provide feedback.

Examples:

* Hover
* Click
* Scroll
* Drag
* Expand
* Progress
* Success
* Achievement unlocks

Visitors should constantly discover small delightful moments.

---

# Cursor

The cursor may change in specific sections.

Examples:

* Magnifying glass
* Coffee mug
* Rocket
* Pencil

These should be optional and never interfere with usability.

---

# Accessibility

Animations should respect reduced motion preferences.

Maintain sufficient color contrast.

Support keyboard navigation.

Use semantic HTML.

Provide focus states.

---

# Responsive Design

The experience should be designed separately for:

Desktop

Tablet

Mobile

Do not simply shrink the desktop layout.

Each breakpoint should feel intentional.

---

# Performance Goals

* Fast initial load
* Lazy-load heavy animations
* Optimize images
* Reuse components
* Avoid unnecessary JavaScript

Interactions should feel smooth even on mid-range devices.

---

# Overall Feeling

If the website were described in one sentence, it should be:

"A playful product experience that makes recruiters smile while clearly demonstrating product thinking, marketing strategy, and problem-solving skills."
